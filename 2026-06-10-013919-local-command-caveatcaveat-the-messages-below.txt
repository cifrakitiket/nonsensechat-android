     Connected, Reconnecting, Error}

     - VpnStateRepository.kt — @Singleton, StateFlow<VpnState> (паттерн как у
     ProxyController).
     - VpnConfig.kt — модель настроек VPN: аутбаунд (VLESS/Reality поля или
     ProxyEntry для SOCKS),
     список split-tunnel приложений, IPv6-тумблер, DNS. Переиспользует парсер из
     ProxyConfig.kt.
     - SingBoxConfigBuilder.kt — чистый Kotlin: VpnConfig → JSON-конфиг sing-box
     (tun-inbound,
     vless/socks-outbound, DNS-over-HTTPS, route-rules). Тестируется без
     нативной либы.
     - TunnelEngine.kt — интерфейс (start(tunFd, configJson, protect) / stop())
     + NoopTunnelEngine
     заглушка. Это ключ к фазовости: весь проект компилируется без libbox.aar.
     - NonsenseVpnService.kt — @AndroidEntryPoint VpnService: onStartCommand →
     startForeground
     с типом specialUse, Builder (addAddress/addRoute
     0.0.0.0+::/setMtu/addDnsServer/
     addDisallowedApplication(self)) → establish(), отдаёт fd в TunnelEngine,
     реализует
     protect() (антипетля), пишет VpnStateRepository, корректный teardown.
     - VpnNotification.kt — канал+нотификация (паттерн
     CallService/Notifications.kt).

     UI в ui/vpn/:
     - VpnViewModel.kt (@HiltViewModel) — состояние из VpnStateRepository, флоу

     connect()/disconnect(), событие-триггер для consent-intent.
     - VpnScreen.kt — большой тумблер коннекта по VpnStage, поле конфига

     SOCKS-прокси»), split-tunnel пикер, DNS, счётчики байт. Переиспользует
     SectionTitle/ToggleRow
     из SettingsScreen.kt.

     <service ... android:permission="android.permission.BIND_VPN_SERVICE"
